Difference between Button and Link

  • Anchor links without href and doesn't navigate should be button
  • Anchor links navigate, a button stays in page like opening modal, dropdown etc
  • Tabindex and click events tacked onto an anchor tag are a markup anti-pattern
  • Buttton is activated with spacebar , links are activated with enter. Eg. PayNow button on invoice page

More read: https://marcysutton.com/links-vs-buttons-in-modern-web-applications/

Debug InAccessible forms

  • Click on the label, focus should go to input field, or checkbox radio button should be selected
  • If there is no visible label, screen should read the label info from aria-label, title, aria-labelledBy
  • Any additonal tooltip information should be spoken i.e. aria-describedby
  • Leave all inputs blank and attempt to submit the form.
    Error messages must be spoken automatically, the screen reader should NOT go silent on a bad submit!
    Focus should be sent to the list of error messages or the first invalid input with an inline error message that's programmatically associated.
    Focus does not have to move but only if the error messages are spoken automatically, e.g. role=alert or aria-live=assertive.

How to debug Modal dialog

Modal dialogs can not be tested with automated tools!

  • Using keyboard only
  • Set focus on the element that opens a modal.
  • Focus is set to dialog or the first focusable element inside the dialog.
  • Cycle through all the elements in the dialog and if it's a Modal then focus should be trapped inside the dialog until the user takes action to close it.
  • Close the dialog using the close button and/or the ESC key.
  • When a modal is closed, focus MUST be sent back to where it came from. Should not start again from address bar
  • Every button in that dialog must have a good accessible name. X is NOT a good accessible name for the close button. Should have aria-label="Close".
